# 学習トラッカースキル

## 概要

学習した技術概念・知識を構造化して蓄積し、復習・振り返りに活用するためのスキル。
「動かしながら学ぶ」スタイルで得た知識を、流さずに資産として残すことが目的。

## 基本思想

- 学習は70%理解したら次へ進む。完璧は目指さない
- ただし学んだことは記録して、後から振り返れるようにする
- 自分の言葉で説明→正誤確認のプロセスを記録に残す

---

## 「理解した」の定義

以下の3段階をクリアして初めて「理解した」とみなす。

1. 自分の言葉で説明できる
2. 「なぜ？」「それで防げるの？」と突っ込まれても答えられる
3. 弱点・限界まで説明できる

---

## データ形式

学習記録は以下のMarkdownファイルに蓄積する。

**ファイルパス:**
- 情報処理安全確保支援士: `~/projects/learning/security-specialist/learning_log.md`
- AI・エージェント・Claude Code・LangChain: `~/projects/learning/ai-agent-sandbox/learning_log.md`
- Web開発（React・TypeScript・Next.js）: `~/projects/learning/web-development/learning_log.md`
- C++ゲーム開発（2d-platformer）: `~/projects/learning/cpp-game-dev/learning_log.md`
- その他プロジェクト: `~/projects/learning/[プロジェクト名]/learning_log.md`

**注意:** プロジェクトごとに独立したlearning_log.mdを管理する

**1エントリの形式：**

```markdown
### [連番] 概念名
- **日付:** YYYY-MM-DD(曜日)  例: 2026-02-19(木)
- **カテゴリ:** C++ / SFML / 設計パターン / ツール / セキュリティ / その他
- **プロジェクト:** どのプロジェクトで学んだか
- **自分の理解:** ユーザーが自分の言葉で説明した内容
- **正誤:** 正解 / 一部修正
- **補足:** Claudeが補足・修正した内容（正解の場合は省略可）
- **実務での使用頻度:** 高 / 中 / 低
- **関連:** 関連するエントリの番号（あれば）
```

## トリガーと動作

### 1. 学習記録の追加

**トリガー:**
- 「学習を記録して」「これ記録しておいて」と言われたとき
- ユーザーが明示的に記録を指示した場合のみ

**自動記録はしない理由:**
- ユーザーが重要だと判断した内容だけを記録する
- ログが膨大になるのを防ぐ
- 意識的な学習記録を促進する（507日ルールと同じ）

**動作:**
1. 現在のチャットまたは会話の文脈から、どのプロジェクトの学習かを判定
   - セキュリティ・暗号・試験対策 → security-specialist
   - AI・エージェント・Claude Code・LangChain → ai-agent-sandbox
   - React・TypeScript・Next.js等のWeb技術 → web-development
   - C++・SFML・ゲーム開発 → cpp-game-dev
2. 該当するlearning_log.mdを読み込み、最新のエントリ番号を確認
3. ユーザーの説明内容と正誤判定を整理する
4. 新しいエントリを追記する
5. 追記内容をユーザーに表示して確認する
6. git commit → git push する（コミットメッセージ例: `docs: 学習ログ追記 - [概念名]`）

### 2. 学習の振り返り

**トリガー:**
- 「何を学んだか振り返りたい」
- 「最近学んだことは？」
- 「C++で学んだことをまとめて」

**動作:**
1. learning_log.md を読み込む
2. 指定された条件（カテゴリ、期間、プロジェクト等）でフィルタリング
3. 要約を表示する

### 3. 過去の学習を検索

**トリガー:**
- 「○○について前に学んだっけ？」
- 「基底クラスってなんだったっけ」

**動作:**
1. learning_log.md からキーワードで検索
2. 該当エントリを表示
3. 必要なら復習として再説明する

### 4. 学習統計

**トリガー:**
- 「学習の進捗を見せて」
- 「どのカテゴリが多い？」

**動作:**
1. learning_log.md を集計
2. カテゴリ別の件数、正解率、期間ごとの学習量を表示

## 記録のルール

- **明示的な記録指示が必要。** ユーザーが「記録して」と言うまで、自動では記録しない
- **記録するのは「正解」と「一部修正」のみ。** 完全な誤解の場合は記録せず、正しい理解ができてから記録する
- **理解度の判定は正直に。** 曖昧な理解を「正解」にしない。補足が必要な場合は「一部修正」とする
- **シンプルに保つ。** 1エントリは簡潔に。長い説明より要点を記録する
- **関連づけを大切に。** 既存のエントリと関連がある場合はリンクする
- **重複を避ける。** 同じ概念を再度学んだ場合は既存エントリに追記する
- **日付フォーマット:** YYYY-MM-DD(曜日) 例: 2026-02-19(木)

---

## ファイルが存在しない場合

該当するlearning_log.md が存在しない場合は、以下のヘッダーで新規作成する。

```markdown
# 学習ログ - [プロジェクト名]

[プロジェクト名]での技術学習で得た知識の蓄積。自分の言葉で説明→正誤確認のプロセスを記録。

---
```

**例:**
- security-specialist: `# 学習ログ - 情報処理安全確保支援士`
- ai-agent-sandbox: `# 学習ログ - AI-Agent-Sandbox`

## 使用例

**例1：学習中に概念を説明した場合**

ユーザー: 「基底クラスって、一番上の親のようなクラスですよね」
Claude: 「ほぼ正解です。補足すると〜」
ユーザー: 「記録しておいて」
→ learning_log.md に追記

**例2：記録指示がない場合**

ユーザー: 「ポインタって、メモリアドレスを保持する変数ですよね」
Claude: 「正解です！」
→ 記録しない（ユーザーが「記録して」と言うまで待つ）

**例3：振り返り**

ユーザー: 「SFMLで学んだことを振り返りたい」
→ カテゴリ「SFML」でフィルタして一覧表示

**例4：検索**

ユーザー: 「AABBって何だったっけ」
→ 過去のエントリを検索して表示し、必要なら復習
