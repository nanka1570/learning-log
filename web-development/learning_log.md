### 1 Next.js App Router のデータフロー
- **日付:** 2026-02-20(金)

- **プロジェクト:** kakeibo-app
- **自分の理解:** Server Component がDBからデータを取得して Client Component に props で渡す。Client Component はそのデータを表示し、ユーザー操作時に Server Action を呼んでDB書き込み（作成・更新・削除）を行う。
- **正誤:** 正解

### 2 マイグレーション（migration）
- **日付:** 2026-02-20(金)
- **カテゴリ:** データベース
- **プロジェクト:** kakeibo-app
- **自分の理解:** マイグレーションは「移行」という意味。ある技術・環境から別の技術・環境に移ること。今回はSQLiteからSupabase PostgreSQLへの移行のこと。
- **正誤:** 正解
- **補足:** Prismaではマイグレーションファイル（migration.sql）がDBの構造変更の手順書として機能し、Gitがコードの変更履歴を管理するように、マイグレーションはDBの変更履歴を管理する。
- **実務での使用頻度:** 高

### 3 Prisma の役割
- **日付:** 2026-02-20(金)
- **カテゴリ:** データベース
- **プロジェクト:** kakeibo-app
- **自分の理解:** PrismaはDB自体ではなく、DB周りをいろいろできるツール。
- **正誤:** 正解
- **補足:** Prismaの3つの役割: (1) スキーマ定義（schema.prismaでテーブル構造を定義）、(2) マイグレーション（スキーマ変更をSQLに変換しDBに適用）、(3) クライアント/ORM（TypeScriptからSQLを書かずにDBを操作）。一番よく使うのはORM部分（prisma.salary.findMany()等）。
- **実務での使用頻度:** 高
