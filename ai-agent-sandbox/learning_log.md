# 学習ログ - AI-Agent-Sandbox

AI・エージェント・Claude Code・LangChain等での技術学習で得た知識の蓄積。自分の言葉で説明→正誤確認のプロセスを記録。

### 001 CI/CD
- **日付:** 2026-02-23(月)
- **カテゴリ:** AI・エージェント / ツール
- **プロジェクト:** AI-Agent-Sandbox（実践ClaudeCode入門 読書）
- **自分の理解:** git pushするコマンドがCLIだと使えるから、AI AgentがCI/CDパイプラインの起点（push）まで担当でき、その後のテスト・ビルド・デプロイが全自動で動く
- **正誤:** 正解
- **補足:**
  - CI（Continuous Integration）: pushされたらサーバー側（GitHub Actions等）で自動テスト・ビルドを実行する仕組み
  - CD（Continuous Delivery/Deployment）: テスト通過後に自動デプロイする仕組み（例: Vercel）
  - CLI型エージェントはコマンドを直接実行できるため、git pushまで担当でき、CI/CDと組み合わせることで「指示だけで本番反映まで全自動」が実現できる
  - hooks（例: Stop hook）はCI/CDとは別物。ローカルでの作業リマインダーや自動チェックに使う
- **実務での使用頻度:** 高
- **関連:** なし

### 002 1stパーティ / 3rdパーティ
- **日付:** 2026-02-23(月)
- **カテゴリ:** AI・エージェント / ツール
- **プロジェクト:** AI-Agent-Sandbox（実践ClaudeCode入門 読書）
- **自分の理解:** 1stパーティは作った会社自身が提供しているもの。Claude CodeはAnthropicが作ったClaude modelをAnthropicが作ったCLIで使えるので1stパーティ。
- **正誤:** 正解
- **補足:**
  - 1stパーティ: 作った本人が提供（例: AnthropicがClaude Codeを提供）
  - 3rdパーティ: 第三者（外部）が提供（例: 他社がClaudeを組み込んだツールを提供）
  - 2ndパーティはソフトウェア業界ではほとんど使われない（ゲーム業界固有の概念）
  - 1stパーティの利点: 最新モデルへ直接アクセス、新機能対応が早い、最適化が深い
- **実務での使用頻度:** 低（概念理解として重要）
- **関連:** なし

### 003 スペック駆動開発
- **日付:** 2026-02-23(月)
- **カテゴリ:** AI・エージェント / 設計パターン
- **プロジェクト:** AI-Agent-Sandbox（実践ClaudeCode入門 読書）
- **自分の理解:** 仕様書を作り、そのドキュメントを中心に開発すること
- **正誤:** 正解
- **補足:**
  - 仕様書を先に書き、それを「唯一の正解」として実装・検証する手法
  - AI Agent 開発で特に重要：AIは曖昧な指示を勝手に解釈するため、詳細な仕様書があるほど解釈ミスが減る
  - 検証基準も「動けばOK」ではなく「仕様書通りか？」になる
  - kakeibo-app の docs/ フォルダ（8文書体系）がスペック駆動開発の実践例
- **実務での使用頻度:** 高
- **関連:** なし

### 004 Claude Code Action
- **日付:** 2026-02-24(火)
- **カテゴリ:** AI・エージェント / ツール
- **プロジェクト:** AI-Agent-Sandbox（実践ClaudeCode入門 読書）
- **自分の理解:** IssueのトリアージができるようになるのでIssueというイベントを起点にしてClaudeを動かすことができる
- **正誤:** 正解
- **補足:**
  - Claude Code ActionはGitHub Actionsのワークフローとして動作する
  - GitHubのイベント（Issue作成、PRオープン、コメント等）を起点にClaudeが自動実行される
  - トリアージとはIssueを読んで、ラベル付け・優先順位付け・担当者アサインなどを自動で行うこと
  - アンビエントエージェントの概念と直結：人間の指示なしにイベント起点でClaudeが動く
- **実務での使用頻度:** 高
- **関連:** 001（CI/CD）、005（アンビエントエージェント）

### 005 アンビエントエージェント
- **日付:** 2026-02-24(火)
- **カテゴリ:** AI・エージェント / 設計パターン
- **プロジェクト:** AI-Agent-Sandbox（実践ClaudeCode入門 読書）
- **自分の理解:** イベントを起点にしてAIに実行させることで、毎回指示を出すより時間が短縮できる
- **正誤:** 一部修正
- **補足:**
  - 時間短縮は正しいが、本質は「人間の認知リソースの解放」
  - 従来：人間が気づいて指示 → AIが実行
  - アンビエント：イベントを自動検知 → AIが実行（人間の介入不要）
  - 追加の価値：複数イベントを並列処理できる／人間が離席中も動く／「あれをやらなきゃ」と覚えておく必要がなくなる
  - Claude Code Actionはアンビエントエージェントの具体的な実装例（GitHubイベント起点）
- **実務での使用頻度:** 高
- **関連:** 004（Claude Code Action）
